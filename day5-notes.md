### NAT 网关（Network Address Translation Gateway）
NAT网关：是阿里云提供的一种企业级公网网关服务，用于实现 VPC 内的云资源通过共享公网 IP 访问互联网，或者向互联网提供服务。
### SNAT（Source Network Address Translation，源地址转换）
SNAT：当 VPC 内的 ECS 主动访问公网时，NAT 网关会将 ECS 的私网源 IP 转换为公网 IP，从而实现多个 ECS 共享少量公网 IP 访问外网。
### 为什么需要 SNAT？
- 节省公网 IP 资源：多台 ECS 共享一个公网 IP 出网
- 安全性：私有子网 ECS 不直接暴露公网 IP，降低被攻击风险
- 统一出口：所有内网机器对外表现为同一个公网 IP，便于外部白名单管理
### 架构
互联网 │ ▼ EIP → NAT网关 → VPC(my-vpc: 10.0.0.0/16) ├── 公网子网 10.0.1.0/24 ├── 应用子网 10.0.2.0/24 ← SNAT 上网 └── 数据库子网 10.0.3.0/24 ← SNAT 上网
### SNAT vs DNAT
| 类型 | 作用 | 方向 |
|------|------|------|
| SNAT | 内网访问外网 | 出方向 |
| DNAT | 外网访问内网 | 入方向 |

# 阿里云 VPC NAT 网关 SNAT 分流对比实验
1. 在阿里云 VPC 创建增强型公网 NAT 网关，绑定 EIP 并配置 SNAT 转发规则，模拟企业内网多主机共享公网访问场景；
2. 通过 VNC 登录 CentOS 云主机，执行curl ifconfig.me获取实例原生弹性公网 IP，搭建流量访问基准；
3. 添加临时精准静态路由ip route add 1.1.1.1 via 网关私网IP，指定目标流量转发至 NAT 网关；
4. 对比两条外网出口访问结果，验证 SNAT 源地址转换机制，掌握 Linux 路由优先级、云网络多出口分流逻辑；
5. 实验完成后按规范删除 SNAT 条目、解绑并释放 NAT 网关与 EIP，管控按量计费资源成本。
## 实验核心知识点
SNAT 作用：内网无独立公网 IP 的服务器，共享 NAT 网关 EIP 访问互联网，节约公网资源；
多出口分流原理：单台 ECS 可存在多条外网出口，精准静态路由优先级高于默认全量路由，可定向分流测试；
临时静态路由ip route add仅内存生效，服务器重启自动清除，无破坏性，适合课堂验证；
阿里云 NAT 网关资源释放约束：存在 SNAT 条目时，EIP 无法解绑，必须先删除转发规则。
