### SLB vs Nginx 负载均衡对比
| 对比项 | Nginx | 阿里云 SLB |
|--------|-------|-----------|
| 层级 | 七层为主 | 四层+七层 |
| 部署 | 软件，装在服务器上 | 云服务，开箱即用 |
| 高可用 | 需要自己做 | 自带高可用 |
| 弹性扩展 | 手动 | 自动弹性 |
| 成本 | 免费软件 | 按量付费 |
| 适用 | 中小型应用 | 企业级应用 |

### 负载均衡算法
| 算法 | 说明 | 适用场景 |
|------|------|---------|
| 轮询 | 依次轮流分发 | 服务器配置相同 |
| 加权轮询 | 按权重比例分发 | 服务器配置不同 |
| 最少连接 | 分给连接数最少的 | 长连接场景 |
| 源IP哈希 | 同一个IP始终分到同一台 | 需要会话保持 |

**: 负载均衡有什么作用？**
1. 高可用：一台挂了，流量自动转到其他正常的
2. 流量分发：把请求分摊到多台服务器，减轻单台压力
3. 水平扩展：随时可以增加/减少后端服务器
4. 健康检查：自动剔除故障节点

**: 四层和七层负载均衡的区别？**
- 四层：基于 IP + 端口，速度快，看不到应用层内容
- 七层：基于 HTTP/HTTPS，能看到 URL、Cookie 等，功能更丰富

**: 常见的负载均衡算法？**
- 轮询、加权轮询、最少连接、源IP哈希、随机等
