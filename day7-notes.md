### 重点知识总结

#### 1. OSI 七层模型
| 层数 | 名称 | 协议/设备 |
|------|------|----------|
| 7 | 应用层 | HTTP、FTP、DNS |
| 6 | 表示层 | 加密、格式转换 |
| 5 | 会话层 | 会话建立与管理 |
| 4 | 传输层 | TCP、UDP |
| 3 | 网络层 | IP、ICMP、路由器 |
| 2 | 数据链路层 | 以太网、交换机 |
| 1 | 物理层 | 网线、光纤 |

#### 2. TCP/IP 四层模型
| 层数 | 名称 | 对应 OSI |
|------|------|---------|
| 4 | 应用层 | 应用+表示+会话 |
| 3 | 传输层 | 传输层 |
| 2 | 网络层 | 网络层 |
| 1 | 网络接口层 | 数据链路+物理 |

#### 3. IP 地址与子网划分
- A类：1.0.0.0 ~ 126.255.255.255，默认掩码 255.0.0.0
- B类：128.0.0.0 ~ 191.255.255.255，默认掩码 255.255.0.0
- C类：192.0.0.0 ~ 223.255.255.255，默认掩码 255.255.255.0
- D类：224.0.0.0 ~ 239.255.255.255（组播）
- E类：240.0.0.0 ~ 255.255.255.255（保留）

**私有地址段：**
- A类：10.0.0.0/8
- B类：172.16.0.0/12
- C类：192.168.0.0/16

#### 4. 以太网帧结构
- 目的 MAC（6字节）
- 源 MAC（6字节）
- 类型/长度（2字节）
- 数据（46~1500字节）
- FCS 校验（4字节）

#### 5. 交换机工作原理
- **学习**：学习源 MAC 地址，构建 MAC 地址表
- **转发**：根据目的 MAC 查找端口，单播转发
- **泛洪**：未知单播帧，向所有端口泛洪
- **老化**：MAC 地址表老化时间（默认 300 秒）

#### 6. 路由原理
- **静态路由**：手动配置，稳定但不灵活
- **动态路由**：协议自动学习，灵活但开销大
- **路由表**：目的网络、子网掩码、下一跳、出接口、优先级

#### 7. OSPF 协议
- 链路状态路由协议
- 组播地址：224.0.0.5（所有路由器）、224.0.0.6（DR/BDR）
- 协议号：89
- 管理距离：10（华为）/ 110（思科）
- 区域划分：骨干区域 Area 0 + 非骨干区域
- DR/BDR 选举：先比优先级，再比 Router ID
- 5种报文：Hello、DBD、LSR、LSU、LSAck

## 📊 传统网络 vs 云上对应关系

| 传统网络 | 阿里云 | 作用 |
|---------|--------|------|
| VLAN | 子网（Subnet） | 网络划分 |
| 交换机 | 虚拟交换机 | 二层转发 |
| 路由器 | 路由表 + NAT网关 | 路由转发 |
| 防火墙 | 安全组 + 网络ACL | 访问控制 |
| NAT / PAT | NAT 网关 SNAT | 内网访问外网 |
| NAT Server | NAT 网关 DNAT | 外网访问内网 |
| 公网IP | 弹性公网IP（EIP） | 公网地址 |
| 负载均衡器 | SLB 负载均衡 | 流量分发 |
| 服务器 | ECS 云服务器 | 计算资源 |

## 💡 学习心得

1. 云上的网络概念和传统网络是通的，只是载体变了
2. 动手实操比光看视频印象深得多
3. 安全组是云上最常用的网络安全手段
4. Nginx 是运维必会的工具，功能很强大
5. 负载均衡是高可用架构的基础
